# Build Skills

<Badge type="warning" text="Intermediate" /> <Badge type="info" text="11 Modules" /> <Badge type="tip" text="Partial Narration" />

Page objects, API testing, auth fixtures, test data, and CI/CD integration. This course bridges the gap between writing tests and building a sustainable test practice.

## Course Overview

Build Skills draws from the legacy curriculum (modules 11–21) to cover intermediate topics that every test automation engineer needs. Topics include prompt engineering, CI/CD, authentication patterns, and accessibility testing.

::: info Content Status
All 11 modules are authored with quizzes and exercises. Some modules are still awaiting narration scripts.
:::

## Module Breakdown

### 11. Prompt Templates
Reusable prompt patterns for Copilot. Structure templates with context, task, and constraints for consistent test generation.

### 12. Reading Results
Deep dive into Playwright's output formats — terminal, HTML report, trace files, JUnit XML. Interpret failures like evidence.

### 13. Human-in-the-Loop Checklist
Quality gates for automated tests. When to trust automation, when to manually verify, and how to blend both approaches.

### 14. Collaborative Test Authoring
Working with developers on test code. PR review patterns, shared page objects, and test ownership models.

### 15. CI/CD Reference
Running Playwright in GitHub Actions. Configuration, artifacts, retries, and parallel execution in pipelines.

### 16. Auth Fixtures
Reusable authentication state with Playwright fixtures. `storageState`, global setup, and role-based test contexts.

### 17. Visual Regression
Screenshot comparison testing. Baseline management, threshold configuration, and handling expected visual changes.

### 18. Accessibility Testing
Automated a11y checks with `@axe-core/playwright`. WCAG compliance, custom rules, and the practice app's intentional violations.

### 19. Flaky Test Diagnosis
Root causes of test flakiness — timing, shared state, network dependencies. Systematic debugging with `--repeat-each` and traces.

### 20. Test Data Strategies
Managing test data — factories, fixtures, API seeding, and cleanup. Avoiding test coupling through data isolation.
